# Corrector BOT

Небольшой Telegram-бот, который исправляет грамматику, орфографию, пунктуацию и типографику в исходящих сообщениях с помощью OpenAI API.

Бот редактирует сообщение на месте. В обычном режиме он сохраняет исходный смысл, стиль, тон и намерение автора. Дополнительные режимы позволяют изменить подачу текста: сделать её более вежливой, формальной, краткой или шуточной.

Если пользователь отвечает на другое сообщение, бот также передаёт модели текст исходного сообщения как контекст. Это помогает точнее понимать местоимения, смысл ответа, тон разговора и выбранную формулировку. Контекст используется только для понимания ответа и не включается в итоговый текст.

## Использование

Добавьте в начало сообщения три точки:

```text
...привет отправь пожалуйста документы до завтра
```

После обработки исходное сообщение будет заменено исправленной версией:

```text
Привет! Отправь, пожалуйста, документы до завтра.
```

Поддерживаются:

- обычные текстовые сообщения;
- подписи к фотографиям, видео и другим медиа;
- дополнительные режимы обработки;
- использование текста сообщения, на которое отвечает пользователь, в качестве контекста.

## Контекст ответов

Когда сообщение отправлено как ответ на другое сообщение, бот передаёт модели два текста:

- текст, который нужно исправить или переписать;
- текст сообщения, на которое отвечает пользователь.

Например, пользователь отвечает на сообщение:

```text
Когда вы сможете отправить подписанный договор?
```

и пишет:

```text
...[diplomat] завтра отправлю
```

Модель получает исходное сообщение как контекст и может сформулировать ответ точнее:

```text
Добрый день! Отправлю подписанный договор завтра.
```

Контекст нужен только для понимания смысла ответа. Бот не редактирует исходное сообщение, не пересказывает его и не добавляет его в результат.

## Режимы

Режим указывается после трёх точек в квадратных скобках:

```text
...[режим] текст сообщения
```

Без указания режима бот выполняет только коррекцию ошибок:

```text
...привет отправь пожалуйста документы
```

### Доступные режимы

| Режим | Назначение |
|---|---|
| `default` | Исправляет ошибки, сохраняя исходные стиль, тон и формулировку |
| `diplomat` | Делает сообщение вежливым, дипломатичным и подходящим для делового общения |
| `joking` | Добавляет лёгкий и дружелюбный юмор |
| `black_joking` | Переписывает сообщение в стиле чёрного юмора |
| `smart_insult` | Превращает грубое оскорбление в литературную аристократическую колкость |
| `formal` | Переписывает сообщение в формальном деловом стиле |
| `short` | Делает текст короче и яснее без удаления существенной информации |

### Примеры

Дипломатичный стиль:

```text
...[diplomat] вы уже неделю не можете отправить договор
```

Возможный результат:

```text
Добрый день! Хотел бы уточнить, когда вы сможете отправить договор, поскольку мы ожидаем его уже неделю.
```

Лёгкий юмор:

```text
...[joking] сервер опять упал
```

Возможный результат:

```text
Сервер опять упал — похоже, стабильность сегодня взяла выходной.
```

Чёрный юмор:

```text
...[black_joking] проект снова не запустился
```

Возможный результат:

```text
Проект снова не запустился — зато надежды окончательно завершили работу без ошибок.
```

Аристократическая колкость:

```text
...[smart_insult] иди нахуй
```

Возможный результат:

```text
Уважаемый сударь, ввиду непреодолимых разногласий предлагаю вам продолжить путь в направлении, традиционно обозначаемом тремя буквами.
```

Формальный стиль:

```text
...[formal] привет документы уже готовы?
```

Возможный результат:

```text
Добрый день. Подскажите, пожалуйста, готовы ли документы?
```

Краткая версия:

```text
...[short] добрый день хотел уточнить получилось ли у вас посмотреть наше предложение
```

Возможный результат:

```text
Добрый день! Удалось посмотреть наше предложение?
```

## Требования

- Python 3.12+
- Telegram-бот, подключённый к аккаунту через функцию «Чат-боты»
- OpenAI API key
- Docker и Docker Compose — необязательно

## Настройка

Создайте файл `.env` в корне проекта:

```dotenv
BOT_TOKEN=your_telegram_bot_token
ADMIN_IDS=[123456789]

OPENAI_API_KEY=your_openai_api_key
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=gpt-5.4-nano-2026-03-17
```

`ADMIN_IDS` содержит Telegram ID пользователей, которым разрешено использовать бота.

Не добавляйте `.env` в Git и не публикуйте реальные токены или API-ключи.

## Запуск через uv

Установите зависимости:

```bash
uv sync
```

Запустите бота:

```bash
uv run python run.py
```

## Запуск через Docker Compose

Соберите и запустите контейнер:

```bash
docker compose -f docker-compose.without-dokploy.yml up -d --build
```

Просмотр логов:

```bash
docker compose -f docker-compose.without-dokploy.yml logs -f
```

Остановка:

```bash
docker compose -f docker-compose.without-dokploy.yml down
```

## Конфиденциальность

В OpenAI API отправляются:

- текст обрабатываемого сообщения;
- текст сообщения, на которое отвечает пользователь, если ответ содержит контекст.

Не используйте бота для отправки:

- паролей;
- API-ключей и токенов;
- платёжных данных;
- персональных или других чувствительных сведений.

Это ограничение также относится к сообщениям, на которые пользователь отвечает, поскольку их текст может быть передан модели как контекст.
